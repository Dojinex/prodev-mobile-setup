# Project Reset Documentation

## Objective

This document explains the purpose and behavior of the `reset-project` command in an Expo Router project. The reset command helps developers clean up demo files and return the project to a minimal, fresh state.

---

## Reset Command Used

```bash
npm run reset-project

What Happens When the Project Is Reset

When the reset-project command is executed, the following actions occur:

Example and demo files are removed

Sample screens and placeholder content generated by Expo are deleted.

This includes demo routes, example components, and instructional UI files.

Project structure is simplified

The app directory is cleaned to retain only essential folders.

The project becomes easier to understand for beginners.

No dependencies are removed

Installed packages in node_modules remain unchanged.

package.json and configuration files are preserved.

Application returns to a clean starting point

The app is ready for custom development without distractions from demo content.

Why the Reset Command Is Important

Helps beginners avoid confusion caused by preloaded demo files

Provides a clean workspace for building real application features

Useful after experimentation or learning exercises

Observation Summary
Aspect	Result
Demo screens	Removed
Routing structure	Cleaned
Dependencies	Preserved
App usability	Improved

Conclusion

The reset-project command is a helpful tool in Expo projects. It allows developers to start fresh without reinstalling dependencies or recreating the project, making it ideal for learning and real-world development.
```
